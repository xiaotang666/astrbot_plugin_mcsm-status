# 更新日志

## v1.9.0 (2026-07-11)
- 修复背景图功能：支持本地背景图（放入 `backgrounds/` 目录即可自动识别，无需配置 URL）
- 修复 `_sync_bg_cache` 不再误删用户手动放入的本地背景图文件
- 新增 `mcsm panel <序号>` 支持数字序号，仅渲染指定实例（如 `mcsm panel 1`）
- 新增 `mcsm say <内容>` 命令，向运行中实例发送 `say` 广播（多实例自动广播所有运行中实例）
- 修复 `mcsm cmd` 多实例场景下第一个词被误当实例名的问题，现在未匹配实例名时会提示选择
- 更新帮助文本和权限命令列表，新增 `say` 命令

## v1.8.1 (2026-07-11)
- 新增实例操作命令（start / stop / restart / kill）支持 `list` 输出的数字序号选取
  - 例如 `mcsm start 1` 直接启动列表中第 1 个实例，无需输入完整名称
  - 序号超出范围时给出友好提示
  - 名称/UUID 模糊匹配方式仍然保留

## v1.8.0 (2026-06-18)
- 重写数据获取策略：CPU/内存数据改为通过 `/api/overview` 接口获取节点系统级数据
- 修正玩家数据解析：`currentPlayers` 为整数类型（人数），不再误处理为列表
- 修正运行时长计算：`started` 字段不是时间戳，改为使用 overview 中的节点 uptime
- 新增多种详情接口端点自动探测（适配不同 MCSM 版本）
- 新增数据归一化层，兼容不同 MCSManager API 响应格式
- 移除不可用的 `/api/protected_instance/info` 详情接口依赖

## v1.7.2 (2026-06-18)
- 新增并发获取运行中实例实时数据（`asyncio.gather`）
- 新增详情接口多端点探测（`protected_instance/info`、`instance/{uuid}` 等）
- 详情接口失败时降级使用列表数据
- 新增运行时数据结构调试日志

## v1.7.1 (2026-06-18)
- 新增暴力递归搜索 API 响应数据树（最大深度 10 层）
- 修正 `_normalize_instance` 字段提取逻辑
- 新增完整的调试数据结构打印

## v1.7.0 (2026-06-18)
- 重构命令注册方式：使用 `@filter.command("mcsm xxx")` + `alias` 注册独立命令
- 新增 `_pre_check` 统一前置检查（API Key、群聊白名单、权限）
- 新增 `_do_action` 实例操作共用逻辑
- 新增版本号和延迟信息显示
- 新增 `_format_bytes` 和 `_format_elapsed` 辅助函数

## v1.6.0 (2026-06-18)
- 新增背景图本地缓存机制（`backgrounds/` 目录）
- URL 移除后自动清理对应缓存文件
- 优化背景图轮询策略（单图内存缓存 + 索引推进）

## v1.5.0 (2026-06-18)
- 新增三级权限控制（超级管理员 / 管理员 / 成员）
- 新增群聊白名单
- 新增自定义背景图 URL 轮询
- 新增面板图片渲染器（Pillow，暗色主题，圆角卡片，进度条）

## v1.0.0 (2026-06-17)
- 初始版本
- 支持 MCSManager 实例状态查询（status / list）
- 支持实例操作（start / stop / restart / kill）
- 支持控制台命令发送（cmd）
- 支持状态面板图片生成（panel）
- 支持中英文命令别名
- 支持实例名称模糊匹配（精确匹配优先）
